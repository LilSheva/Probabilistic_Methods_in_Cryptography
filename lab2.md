# Построение запрета

### Коротко

1. По заданной функции строим таблицу истинности.
2. Каждую строку делим на две строки, дописывая в левый столбец (где были входы) в одну строку 0, а в другую 1 (как бы смотрим, что получится, если следующим входом подадим в функцию разные варианты).
3. Для каждой получившейся строки смотрим в левой колонке правые $n$ цифр (где $n$ — количество аргументов функции: в реальной лабе 6, в примере ниже 3). Подставляем их в функцию, дописываем выход в правую колонку. Теперь в правой колонке 2 цифры: изначальная из таблицы истинности и новая дописанная.
4. Ищем наиболее редко встречающиеся комбинации в правой колонке. Их оставляем, остальные строки вычёркиваем и дальше не рассматриваем.
5. Повторяем шаги 2-4: каждую строку делим пополам, дописываем в левые колонки 0 и 1, подставляем правые $n$ символов из левой колонки в формулу, результат дописываем в правую колонку. Ищем самую (или самые) редко встречаемые комбинации, остальные вычёркиваем.
6. Повторяем до тех пор, пока не получим таблицу, где вся правая колонка будет состоять из одинаковых комбинаций (например, на конце дописали ноль). Тогда дописываем в выходную последовательность другой символ (для примера выше единицу) — это и есть наш запрет.

### Пример

Возьмём регистр сдвига.
$f(\vec x)= x_1x_3\oplus x_2$

| $x_1..x_3$ | $y_1$ |

| 000 | 0 |
| 001 | 0 |
| 010 | 1 |
| 011 | 1 |
| 100 | 0 |
| 101 | 1 |
| 110 | 1 |
| 111 | 0 |

Допишем к каждому входу 0 и 1 (в 2 раза больше строк). Для расчёта нового выхода смотрим на три правых входа (левый бит входа отсекаем).
$f(\vec x)= x_1x_3\oplus x_2$

| $x_1..x_4$ | $y_1~y_2$ |
| --- | --- |
| 0000 | 00 |
| 0001 | 00 |
| 0010 | 01 |
| 0011 | 01 |
| 0100 | 10 |
| 0101 | 11 |
| 0110 | 11 |
| 0111 | 10 |
| 1000 | 00 |
| 1001 | 00 |
| 1010 | 11 |
| 1011 | 11 |
| 1100 | 10 |
| 1101 | 11 |
| 1110 | 01 |
| 1111 | 00 |

Посмотрим, сколько раз встречается каждый выход (правый столбец):
* `00` — 5 раз
* `01` — 3 раза — берём самую редкую комбинацию
* `10` — 3 раза 
* `11` — 5 раз

Разная встречаемость (не 4 4 4 4) — значит, функция не сильно равновероятная.

Далее буду записывать две идентичных таблицы: одну сокращённую, с новыми битами, и одну полную, со всей последовательностью входов и выходов.

Рассмотрим для 01 следующий шаг. Возьмём только строки с выходом 01.

### Задание 1. Преобладание нулей над единицами

Число аргументов $n = 6$. Общее число входных наборов $2^n = 64$.
Все сгенерированные координатные функции сбалансированы:
$$||f_1|| = ||f_2|| = ||f_3|| = ||f_4|| = ||f_5|| = ||f_6|| = 32$$

Рассчитаем преобладание нулей над единицами по формуле:
$$\delta(f) = 1 - 2P(f=1) = 1 - \frac{||f||}{2^{n-1}}$$

Для всех шести координатных функций:
$$\delta(f_i) = 1 - \frac{32}{2^5} = 1 - \frac{32}{32} = 0$$

Преобладание равно $0$, то есть нулей и единиц в векторах значений поровну.

### Задание 2. Сильная равновероятность функций

Функция является *сильно равновероятной*, если в генерируемой ею последовательности выходов отсутствуют запреты (то есть вероятность появления любого выходного кортежа длины $L$ строго положительна: $P(Y^L) > 0$ при любых $L$).

В нашем случае **ни одна из координатных функций не является сильно равновероятной**, так как для каждой из них существует по крайней мере один запрет (выходная последовательность, появление которой физически невозможно из-за структуры регистра сдвига и функции обратной связи).
